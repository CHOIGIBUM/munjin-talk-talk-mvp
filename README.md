# 문진톡톡 (MoonjinTalkTalk)

> 지역 어르신의 말이 의사의 1장 리포트가 되기까지
> 고령 환자의 사투리·구어체 증상과 진료 질문을, 의료진이 바로 읽을 수 있는 **진료 전 리포트**로 정리해주는 AI 문진 서비스

```
어르신 음성  →   AI 정리   →   의사용 리포트
말로 증상·걱정 입력  →  증상·질문 구조화  →  진료 전 1장 전달
```

---

## 프로젝트 개요

**문진톡톡**은 고령 환자가 음성으로 남긴 증상·걱정·진료 질문을 AI가 정리하여 의료진에게 전달하는 **진료 전 문진 서비스**입니다.

환자는 평소 사투리 말투로 말하면 되고, 의료진은 진료 전에 1장 리포트로 환자 상태와 확인할 질문을 빠르게 파악할 수 있습니다.

> 문진톡톡은 진단·처방을 제공하지 않습니다. 의료진 확인을 위한 문진 정보를 정리하는 **인수인계 도구**입니다.

---

## 해결하려는 문제

고령 환자의 진료 현장에는 세 가지 구조적 문제가 있습니다.

| 문제 | 설명 |
|------|------|
| **증상 전달 실패** | 사투리·구어체 표현, 시작 시점·동반 증상 누락으로 의료진이 다시 묻는 과정이 발생 |
| **질문 누락** | 어르신은 진료실에서 복약 걱정, 건강식품, 지인에게 들은 말을 잊고 넘어가기 쉬움 |
| **쓰레기 정보 노출** | 유튜브·인터넷 건강정보가 근거 없이 확산되지만, 환자는 이를 구분하기 어려움 |

배경 데이터:

- **40.3%** — 65세 이상 적절한 건강정보 이해 능력 (성인 전체 60.4% 대비 낮음)
- **62.5%** — 암·당뇨 유튜브 영상 중 근거 없는 비율 (309개 영상 분석)
- **25.7%** — 강원 65세 이상 인구 비중 (전국 20.3%보다 높은 고령화 지역)

## 타겟 사용자와 가치

| 사용자 | Pain Point | 문진톡톡의 가치 |
|--------|-----------|----------------|
| **고령 환자 (강원 사투리)** | 증상과 질문을 진료실에서 빠짐없이 말하기 어려움 | 평소 말투로 말하면 핵심 내용이 정리됨 |
| **지역 의료진 (공보의)** | 짧은 진료 시간 안에 환자 상태와 질문을 파악해야 함 | 진료 전 1장 리포트로 증상과 질문 확인 |
| **보호자 / 간호사** | 환자의 걱정과 질문을 문진 시 대신 정리해 전달해야 함 | 문진/질문 내용을 자동으로 손실 없이 정리 |

---

## 핵심 기능 (5가지)

1. **음성 입력** — 어르신이 버튼 하나로 증상과 질문을 입력
2. **증상 정리** — 사투리·구어체 표현을 표준 증상 표현으로 변환
3. **짧은 후속 질문** — 시작 시점, 동반 증상, 복용 약처럼 빠진 정보를 짧게 질문
4. **진료 질문 정리** — 유튜브·지인·민간요법·복약 걱정을 의사에게 물어볼 질문으로 정리
5. **의사용 리포트** — 증상, 질문, 확인 필요 항목을 1장으로 정리

> AI는 의료적 답을 주는 대신, 어르신의 말을 **증상 · 질문 · 확인 항목**으로 정리합니다.

---

## 워크플로우

```
[환자 음성 입력]  →  [AI 증상 정리]  →  [후속 질문]  →  [진료 질문 정리]  →  [의사용 1장 리포트]
   말로 입력           표준 증상 매핑        빠진 정보 보완       질문 목록 변환          진료 전 30초 확인
```

## 사용 예시

**어르신 발화 (입력)**

> "어제부터 가심이 답답하고 숨이 좀 차요. 밤에 더 그래요.
> 혈압약 먹는데 처방약이랑 계속 먹어야 하는지도 물어보고 싶고,
> 유튜브에서 양파즙이 혈관에 좋다던데 먹어도 되는지 궁금해요."

**문진톡톡 정리 결과**

| 입력 내용 | 정리 결과 |
|-----------|-----------|
| "가심이 답답하다" | 흉부 답답함 |
| "숨이 차다" | 호흡곤란 |
| "밤에 더 그래요" | 야간 악화 |
| "혈압약 계속 먹어야 하나" | 복약 지속 여부 질문 |
| "양파즙 먹어도 되나" | 건강식품 병용 가능 여부 질문 |

**의사용 요약 (출력)**

> 74세 환자. 어제부터 흉부 답답함과 호흡곤란 호소. 야간 악화 언급.
> 혈압약 복용 지속 여부와 건강식품 병용 가능 여부를 질문하고 싶어함.

## SMART 원페이퍼 리포트

의사가 30초 안에 확인할 수 있도록 **S · M · A · R · T** 구조로 구성됩니다.

- **S**ymptom — 환자 원문
- **M**apping — 표준 증상 매핑
- **A**sk — 환자가 묻고 싶은 질문
- **R**eview — 의료진 확인 필요 항목
- **T**ranscript — EMR 복사문

## 경쟁 서비스와의 차별점

| 구분 | 기존 AI 문진/증상 체커 | **문진톡톡** |
|------|-----------------------|-------------|
| 주 사용층 | 일반 환자, 병원 이용자 | 고령 지역 환자, 지역 의료진 |
| 입력 방식 | 문진표, 앱 입력, 텍스트 중심 | **음성 중심** |
| 핵심 목적 | 증상 체크, 예상 질환, 진료 연결 | 증상·질문을 의사에게 전달 |
| AI 역할 | 분석, 추천, 예측 | **정리, 변환, 누락 방지** |
| 최종 결과 | 사용자 안내 또는 병원 업무 자동화 | **의사용 1장 인수인계 리포트** |

> 차별점: 고령층 음성 입력 + 사투리·구어체 증상 정리 + 진료 질문 제공

## MVP 범위

대표 시나리오 하나를 끝까지 구현합니다.

**MVP 시나리오**
> 고령 환자가 흉부 답답함과 호흡곤란을 말함 → 증상 정리 → 복용약·진료 질문 추가 확인 → 의사용 1장 리포트 생성

**구현 범위**

| 단계 | 범위 |
|------|------|
| 입력 | 음성 또는 텍스트 입력 |
| 증상 정리 | 강원 방언/구어체 예시 5~10개 처리 |
| 후속 질문 | 고정 질문 3개 |
| 질문 정리 | 환자 발화를 질문 목록으로 변환 |
| 출력 | 웹 화면 또는 PDF 형태의 1장 리포트 |

> MVP 목표: 완전한 의료 AI 서비스가 아니라, **'어르신의 말이 의사용 리포트로 바뀌는 핵심 가치'를 데모로 증명**합니다.

## 기대 효과

- **어르신** — 진료 때 말하지 못하거나 까먹는 질문을 줄임
- **의료진** — 진료 전 증상과 질문을 빠르게 확인
- **보호자** — 부모님의 걱정과 진료 질문을 함께 정리
- **지역 의료** — 짧은 진료 시간 안에서 문진 품질을 높임

## 면책 조항

문진톡톡은 의료적 진단이나 처방을 제공하지 않습니다. 본 서비스는 환자의 발화를 의료진이 확인하기 쉬운 형태로 **정리**하는 인수인계 도구이며, 모든 의료적 판단은 의료진이 수행합니다.
